FOLLOW-UPS (not fixed during gate, flagged for Phase 10.x or follow-on):

F-1: MIGRATION AUTOMATION
  - argus binary lacks `migrate` subcommand; `make db-migrate` falls through to serve.
  - Pre-existing issue NOT specific to Phase 10, but noticed because phase-10 adds
    new migrations that never run on a fresh deploy.
  - Recommendation: add cobra-based migrate subcommand or run golang-migrate
    in Dockerfile entrypoint.

F-2: PARTITIONED-TABLE MIGRATIONS
  - Multiple migrations fail on partitioned sims/audit_logs (CONCURRENTLY not supported
    on partitioned parents; hypertables-with-columnstore reject RLS).
  - Pre-existing. Not a Phase 10 regression.
  - Recommendation: guard these index creations behind pg_class checks or use
    PARTITION-aware syntax.
