## Fix #1 — SMS History page DTO mismatch

**Surface**: `http://localhost:8084/sms`
**Symptom**: "CREATED" column rendered "Invalid Date" on every row. "PRIORITY" column displayed empty badges.
**Scope framing**: NOT a STORY-086 regression. STORY-086 only recovered the `sms_outbound` table + API; the
/sms UI column binding was always wrong since STORY-069 shipped the SMS Gateway — it only became visible
now because STORY-086 unblocked the endpoint from its 500 state and populated the list with actual rows.
Closing this here under the Phase 10 zero-deferral banner rather than filing a STORY-069 follow-up.

**Root cause**: Pre-existing STORY-069-era UI↔API contract drift. API DTO (internal/api/sms/handler.go:295-308
`smsOutboundDTO`) returns `queued_at`, `status`, `sim_id`, `msisdn`, `text_hash`, `text_preview`,
`provider_message_id?`, `error_code?`, `sent_at?`, `delivered_at?`. Frontend type (web/src/hooks/use-sms.ts
SMSOutbound) declared `created_at`, `priority`, `tenant_id` — none of which exist in the server response.

**Fix applied** (code edits, NOT committed):
1. `web/src/hooks/use-sms.ts:7-19` — SMSOutbound interface rewritten to match API DTO exactly:
   - Removed: tenant_id, priority, created_at (none returned by backend)
   - Added: text_hash (required), delivered_at (optional)
   - Changed: sent_at string|null → string? (optional, matches omitempty semantics)
   - Changed: provider_message_id/error_code from string|null → string?
2. `web/src/pages/sms/index.tsx:155-189` — SMS History table updated:
   - Column header "CREATED" → "QUEUED"
   - Column binding `m.created_at` → `m.queued_at`
   - "PRIORITY" column removed (field not in API response); empty-state colSpan 6 → 5

**Type-check**: `cd web && npx tsc --noEmit` → clean
**Build**: `npm run build` → 2825 modules transformed, built in 4.30s (no errors)
**Post-fix verification**: Rebuilt argus image + restarted container; /sms page now renders
"QUEUED" column with proper "4/17/2026, 10:42:11 AM" timestamps and no "Invalid Date" values.
