# Glossary — Argus

## AAA & Protocol Terms

| Term | Definition | Context |
|------|-----------|---------|
| AAA | Authentication, Authorization, and Accounting | Core function of Argus L1 engine |
| RADIUS | Remote Authentication Dial-In User Service (RFC 2865/2866) | Primary AAA protocol, UDP-based |
| Diameter | Next-gen AAA protocol (RFC 6733) | Used for Gx (policy) and Gy (charging) with operators |
| RadSec | RADIUS over TLS (RFC 6614) | Encrypted RADIUS transport |
| Gx | Diameter interface for policy and charging control | Communicates QoS/policy rules to P-GW |
| Gy | Diameter interface for online charging | Real-time charging/quota management |
| CoA | Change of Authorization (RFC 5176) | Dynamically modify active sessions (e.g., apply new policy) |
| DM | Disconnect Message (RFC 5176) | Force-terminate an active session |
| EAP-SIM | Extensible Authentication Protocol for SIM | Authenticates using GSM SIM credentials |
| EAP-AKA | EAP Authentication and Key Agreement | Authenticates using USIM (3G/4G) credentials |
| EAP-AKA' | Enhanced EAP-AKA for 5G | Adds key separation for 5G networks |
| SBA | Service-Based Architecture | 5G core network architecture using HTTP/2 APIs |
| AUSF | Authentication Server Function | 5G core network function for authentication |
| UDM | Unified Data Management | 5G core network function for subscriber data |
| NRF | NF Repository Function | 5G core network function for service discovery and registration. Argus implements real 3GPP NRF HTTP calls: NFRegister (PUT), NFUpdate (PATCH), NFDeregister (DELETE), NFStatusSubscribe (POST). Activated when `SBA_NRF_URL` env var is set; falls back to no-op when unset. | SVC-04, STORY-020, STORY-063 |
| SUPI | Subscription Permanent Identifier | Globally unique 5G subscriber identifier (typically `imsi-{IMSI}`). Permanent identity resolved from SUCI during authentication. | 5G SBA (TS 23.501), STORY-020 |
| SUCI | Subscription Concealed Identifier | Privacy-preserving encrypted form of SUPI transmitted over the air in 5G. Format: `suci-{MCC}{MNC}{RoutingID}{...}`. Resolved to SUPI by AUSF/UDM. | 5G SBA (TS 23.003), STORY-020 |
| S-NSSAI | Single Network Slice Selection Assistance Information | Identifier for a 5G network slice, composed of SST (Slice/Service Type, 1 byte) and optional SD (Slice Differentiator, 3 bytes). Used for slice-level authentication and policy. | 5G SBA (TS 23.501), STORY-020 |
| 5G-AKA | 5G Authentication and Key Agreement | Primary authentication method for 5G SA networks. Flow: SUCI -> SUPI resolution -> AV generation (RAND/AUTN/XRES*/KAUSF) -> HXRES* verification -> KSEAF derivation. | 5G SBA (TS 33.501), STORY-020 |
| KSEAF | Key for SEcurity Anchor Function | Key derived from KAUSF during 5G-AKA authentication, used as the anchor key for all subsequent 5G security key derivations. Derived via HMAC-SHA256. | 5G SBA (TS 33.501), STORY-020 |
| KAUSF | Key for Authentication Server Function | Master key generated during 5G-AKA authentication. Used to derive KSEAF. Stored temporarily in AUSF during authentication flow. | 5G SBA (TS 33.501), STORY-020 |
| NAS | Network Access Server | Device that sends RADIUS/Diameter requests to Argus |
| PCRF | Policy and Charging Rules Function | LTE network element; Argus implements a mini-PCRF |
| PCEF | Policy and Charging Enforcement Function | Enforcement point in P-GW; receives rules from PCRF |
| Authentication Vector | Set of cryptographic values (RAND, SRES, Kc for GSM; RAND, AUTN, XRES, CK, IK for UMTS/LTE/5G) used in SIM authentication challenge-response. Generated by HLR/AuC or mock adapter. | EAP-SIM/AKA (STORY-016, STORY-018) |
| Triplet | GSM authentication vector containing RAND (16B), SRES (4B), Kc (8B). Used by EAP-SIM. | EAP-SIM, mock adapter vector generation |
| Quintet | UMTS/LTE/5G authentication vector containing RAND (16B), AUTN (16B), XRES (8B), CK (16B), IK (16B). Used by EAP-AKA/AKA'. | EAP-AKA/AKA', mock adapter vector generation |
| MSK | Master Session Key — 64-byte key derived during EAP authentication, split into MS-MPPE-Send-Key (0:32) and MS-MPPE-Recv-Key (32:64) for RADIUS encryption. Derived via HMAC-SHA1 (EAP-SIM) or HMAC-SHA256 (EAP-AKA/AKA'). | EAP-SIM/AKA/AKA' (STORY-016) |
| EMSK | Extended Master Session Key — additional 64-byte key derived alongside MSK. Reserved for future use (e.g., fast re-authentication). | EAP-SIM/AKA/AKA' (RFC 4186/4187/5448) |
| MS-MPPE | Microsoft Point-to-Point Encryption — RADIUS vendor-specific attributes (type 16/17, vendor 311) carrying encrypted session keys derived from MSK. Used in Access-Accept to provide link encryption keys. | RADIUS + EAP (RFC 2548), STORY-016 |
| EAP State Machine | Stateful handler managing multi-round-trip EAP authentication flows. States: identity, sim_start, method_negotiation, challenge, success, failure. Pending challenges stored in Redis with 30s TTL. | SVC-04, STORY-016 |
| Vector Cache | Redis list-based pre-fetch cache for authentication vectors. Key pattern `eap:vectors:{imsi}:{type}`, 5min TTL, batch size 3. Uses LPOP/RPUSH for atomic consume-one-keep-rest semantics. | SVC-04, STORY-016 |
| CER/CEA | Capabilities Exchange Request/Answer — Diameter handshake to negotiate supported applications between peers | Diameter protocol (RFC 6733), STORY-018 adapter |
| DWR/DWA | Device Watchdog Request/Answer — Diameter keepalive mechanism to detect peer failures | Diameter protocol (RFC 6733), STORY-018 adapter |
| DPR/DPA | Disconnect Peer Request/Answer — Diameter graceful peer disconnection. Initiating peer sends DPR with Disconnect-Cause; receiving peer responds with DPA and closes the transport connection. | Diameter protocol (RFC 6733), STORY-019 |
| CCR/CCA | Credit-Control Request/Answer — Diameter message pair for credit-control sessions (command code 272). Used in both Gx (policy) and Gy (charging) interfaces. CC-Request-Type: Initial (1), Update (2), Termination (3), Event (4). | Diameter Gx/Gy (RFC 4006), STORY-019 |
| RAR/RAA | Re-Auth Request/Answer — Diameter message pair for server-initiated re-authorization (command code 258). Used to push mid-session policy changes from PCRF to PCEF. | Diameter Gx (RFC 6733), STORY-019 |
| PCC Rules | Policy and Charging Control rules — QoS and charging rules installed on the PCEF via Gx Charging-Rule-Install AVP. Include QoS-Information (max bandwidth), Flow-Description, and precedence. | Diameter Gx (3GPP TS 29.212), STORY-019 |
| OCS | Online Charging System — Real-time charging function implementing the Gy interface. Grants credit quotas (Granted-Service-Unit), receives usage reports (Used-Service-Unit), and manages prepaid account balances. | Diameter Gy (3GPP TS 32.299), STORY-019 |
| Diameter Session-Id | Globally unique identifier for a Diameter session, formatted as `{DiameterIdentity};{high32};{low32};{optional}`. Maps to `acct_session_id` in TBL-17 sessions table. Shared correlation key across Gx and Gy interfaces. | Diameter protocol (RFC 6733), STORY-019 |
| Diameter Peer | Remote Diameter node connected via TCP to the Argus Diameter server. Peer state transitions: idle (connected, no CER) -> open (CER/CEA complete) -> closing (DPR sent) -> closed. Managed in server.go via sync.Map. | Diameter protocol (RFC 6733), STORY-019 |

## SIM & Mobile Terms

| Term | Definition | Context |
|------|-----------|---------|
| SIM | Subscriber Identity Module | Physical or embedded chip for network authentication |
| eSIM | Embedded SIM (eUICC) | Programmable SIM, profiles downloadable over-the-air |
| eUICC | Embedded Universal Integrated Circuit Card | Hardware chip that hosts eSIM profiles |
| IMSI | International Mobile Subscriber Identity | Unique subscriber identifier (up to 15 digits) |
| MSISDN | Mobile Station International Subscriber Directory Number | Phone number associated with SIM |
| ICCID | Integrated Circuit Card Identifier | Unique SIM card serial number (up to 22 digits) |
| EID | eUICC Identifier | Unique identifier for eSIM hardware (32-digit eUICC chip serial) |
| MCC | Mobile Country Code | 3-digit code identifying a country in ITU E.212 (e.g. 286 = Turkey) |
| MNC | Mobile Network Code | 2-3 digit code identifying an operator within a country (combined with MCC forms PLMN) |
| IMEI | International Mobile Equipment Identity | Device hardware identifier |
| SM-DP+ | Subscription Manager - Data Preparation+ | Server that prepares and delivers eSIM profiles (SGP.22) |
| OTA | Over-The-Air | Remote SIM management via APDU commands |
| APDU | Application Protocol Data Unit | Commands sent to SIM card for configuration. Argus supports 5 command types: UPDATE_FILE, INSTALL_APPLET, DELETE_APPLET, READ_FILE, SIM_TOOLKIT. Built by `ota.BuildAPDU()` in `internal/ota/apdu.go`. | SVC-03, STORY-029, TBL-26 |
| SMS-PP | SMS Point-to-Point | OTA delivery channel using GSM 03.48 SMS envelope to transport APDU commands to the SIM card. Envelope includes security headers (SPI, KIC, KID), Toolkit Application Reference (TAR), counter (CNTR), optional MAC, and secured APDU data. Max 140 bytes per envelope. | SVC-03, STORY-029, `internal/ota/delivery.go` |
| BIP | Bearer Independent Protocol | OTA delivery channel using TCP-based transport for APDU commands. Alternative to SMS-PP for devices with data connectivity. Framing: channel_id (1B) + transport (1B) + port (2B) + data_length (2B) + data. | SVC-03, STORY-029, `internal/ota/delivery.go` |
| KIC | Key for Integrity and Ciphering | 16-byte AES key used for encrypting OTA APDU data via AES-128-CBC. Part of the OTA security framework (GSM 03.48). Used in `kic` and `kic_kid` security modes. | STORY-029, `internal/ota/security.go` |
| KID | Key for Integrity Data | 16-byte key used for computing MAC (HMAC-SHA256, truncated to 8 bytes) over OTA APDU data. Provides message integrity verification. Used in `kid` and `kic_kid` security modes. | STORY-029, `internal/ota/security.go` |
| GSM 03.48 | GSM Technical Specification 03.48 | Standard for Secure Remote SIM Toolkit (STK) messaging. Defines the SMS-PP envelope format for OTA commands including Security Parameter Indicator (SPI), ciphering/MAC key indicators, Toolkit Application Reference (TAR), and replay-protection counter (CNTR). | STORY-029, OTA delivery |
| TAR | Toolkit Application Reference | 3-byte identifier in GSM 03.48 SMS-PP envelope specifying the target application on the SIM card (e.g., SIM Toolkit applet). Used to route OTA commands to the correct on-card application. | STORY-029, `internal/ota/delivery.go` |
| MNO | Mobile Network Operator | Turkcell, Vodafone, TT Mobile |
| MVNO | Mobile Virtual Network Operator | Virtual operator using MNO infrastructure |
| SGP.02 | GSMA M2M eSIM specification | Legacy M2M remote provisioning (push model) |
| SGP.22 | GSMA Consumer eSIM specification | Consumer eSIM with SM-DP+ (pull model) |
| SGP.32 | GSMA IoT eSIM specification | New IoT-specific eSIM standard (2023) |
| eSIM Profile State Machine | Lifecycle states for an eSIM profile on TBL-12 (4 states): `available` (default, loaded on SIM, inactive, eligible to enable) → `enabled` (active; exactly one per SIM enforced by partial unique index) → `disabled` (operator-deactivated, requires manual re-enable) ↔ `available` (switch displaces current profile back to `available`, not `disabled`). `deleted` = soft-deleted (not allowed when `enabled`; returns 409 CANNOT_DELETE_ENABLED_PROFILE). Max 8 profiles per SIM (GSMA SGP.22 cap, enforced by Create handler). CHECK constraint: `profile_state IN ('available','enabled','disabled','deleted')`. State transitions enforced with FOR UPDATE row locks in PostgreSQL transactions. | SVC-03, STORY-028, STORY-061, TBL-12 |
| Profile Switch (eSIM) | Atomic operation that transitions the old (currently `enabled`) eSIM profile to `available` state (not `disabled` — per DEV-164, a switch is a swap, not operator deactivation) and enables a different profile on the same SIM, all in a single PostgreSQL transaction. Post-commit side effects: dispatches DM (RFC 5176) for active sessions; releases old IP address from pool; clears `sims.policy_version_id` for policy re-resolution. Updates `sims.apn_id = NULL` (requires reassignment). Records change in `sim_state_history` (TBL-11). Publishes `esim.profile.switched` NATS event with `{sim_id, old_profile_id, new_profile_id, new_operator_id, timestamp}`. | SVC-03, STORY-028, STORY-061, API-074 |
| SM-DP+ Adapter | Interface for communicating with GSMA SM-DP+ servers (SGP.22) for remote eSIM profile provisioning. 4 methods: DownloadProfile, EnableProfile, DisableProfile, DeleteProfile. Two implementations: `MockSMDPAdapter` (development) and `HTTPSMDPAdapter` (generic SGP.22 ES9+ HTTP provider, active when `ESIM_SMDP_PROVIDER=generic`). | SVC-03, STORY-028, STORY-063, `internal/esim/smdp.go` |
| Profile Load | Operation that adds a new eSIM profile to a SIM card via SM-DP+ DownloadProfile call. Creates a new row in `esim_profiles` in `available` state. Limited to 8 profiles per SIM (GSMA SGP.22 cap). Exposed via API-075 POST /api/v1/esim-profiles. | SVC-03, STORY-061, TBL-12, API-075 |
| Multi-profile eSIM | eSIM SIM card capable of holding multiple profiles simultaneously, with exactly one `enabled` at a time. Argus supports up to 8 profiles per SIM, stored as multiple rows in `esim_profiles` (TBL-12). Partial unique index on `(sim_id) WHERE profile_state='enabled'` enforces the single-active constraint at the DB level. Supersedes the single-profile-per-SIM model from STORY-028 (DEV-088). | SVC-03, STORY-061, TBL-12 |

## Network Terms

| Term | Definition | Context |
|------|-----------|---------|
| APN | Access Point Name | Network entry point identifier; determines routing and services |
| Private APN | Operator-defined APN for enterprise traffic isolation | Core Argus management target |
| P-GW | Packet Data Network Gateway (4G) | Routes traffic based on APN, enforces policy |
| GGSN | Gateway GPRS Support Node (3G) | Legacy equivalent of P-GW |
| RAT | Radio Access Technology | NB-IoT, LTE-M, 4G LTE, 5G NR |
| RAT Type (Canonical) | Normalized string identifier for Radio Access Technology used internally. Canonical values: `utran` (3G), `geran` (2G), `lte` (4G), `nb_iot` (NB-IoT), `lte_m` (LTE-M/Cat-M1), `nr_5g` (5G SA), `nr_5g_nsa` (5G NSA), `unknown`. Protocol-specific values (RADIUS numeric AVP, Diameter Unsigned32, 5G SBA strings) are mapped to canonical form via the `rattype` package. Display names (2G/3G/4G/5G) mapped via `rattype.DisplayName()`. | SVC-04, STORY-027, `internal/aaa/rattype/` |
| NB-IoT | Narrowband IoT | Low-power, low-bandwidth cellular for IoT sensors |
| LTE-M | LTE for Machines (Cat-M1) | Medium-bandwidth cellular for IoT with mobility |
| SoR | Steering of Roaming | Directing SIM to preferred network when multiple available. Argus SoR engine evaluates IMSI prefix, RAT preference, cost, and circuit breaker state to select optimal operator. |
| SoR Decision | Output of SoR evaluation: primary operator, fallback list, selection reason (imsi_prefix_match, cost_optimized, rat_preference, manual_lock, default), and evaluation metadata. Cached in Redis per SIM with configurable TTL (default 1h). Key: `sor:result:{tenant_id}:{imsi}`. | SVC-06, STORY-026 |
| SoR Priority | Integer field on operator_grants (TBL-06) controlling operator preference order in SoR evaluation. Lower value = higher priority. Default 100. | SVC-06, STORY-026, TBL-06 |
| Operator Lock | Per-SIM override stored in `sims.metadata.operator_lock` (UUID) that bypasses SoR evaluation and forces routing to a specific operator. Reason: `manual_lock`. | SVC-06, STORY-026 |
| IMSI Prefix Routing | SoR routing method that matches the first 5-6 digits of an IMSI (MCC+MNC) against operator definitions to determine preferred operator(s). Uses `strings.HasPrefix` for O(1) matching. | SVC-06, STORY-026, F-021 |
| Cost-Based Selection | SoR tie-breaking strategy that selects the operator with lowest `cost_per_mb` when multiple operators have equal priority and RAT rank. | SVC-06, STORY-026, F-038 |
| Roaming Agreement | Tenant-scoped contract between the platform operator and a partner operator defining cost, SLA terms, validity period, and auto-renewal behaviour. Stored in `roaming_agreements` (TBL-43). At most one `active` agreement may exist per (tenant, operator) pair — enforced by partial unique index. States: `draft`, `active`, `expired`, `terminated`. Types: `national`, `international`, `mvno`. | SVC-06, STORY-071, TBL-43 |
| Agreement State | FSM for a roaming agreement: `draft` (created, not yet in force) → `active` (current, in force; cost_per_mb overrides SoR) → `expired` (end_date passed, auto-expiry by cron or manual state patch) → `terminated` (explicitly cancelled; terminated_at timestamp set). Only one `active` agreement per (tenant, operator) is allowed. A terminated agreement cannot be updated or re-activated. | SVC-06, STORY-071, TBL-43 |
| SLA Terms (Roaming) | JSONB sub-object on `roaming_agreements` capturing service-level commitments with a partner operator: `uptime_percent` (required float), `latency_p95_ms` (required int), `max_incidents_per_month` (optional int). Stored verbatim; informational only (not auto-enforced in SoR). | STORY-071, TBL-43 |
| Cost Terms (Roaming) | JSONB sub-object on `roaming_agreements` capturing commercial terms: `cost_per_mb` (required float, overrides SoR default when agreement is active), `currency` (required ISO-4217 string), `tiers` (optional volume-discount tiers), `settlement_period` (optional string). When an active agreement exists for the selected operator, the SoR engine replaces the operator's default `cost_per_mb` with this value and sets `decision.AgreementID`. | SVC-06, STORY-071, TBL-43 |
| SoR Agreement Hook | Extension point in the SoR engine (`internal/operator/sor/engine.go`) that consults an injected `RoamingAgreementProvider` interface during `Evaluate()`. When an active agreement is found for the selected operator, `CostPerMB` on the decision is overridden and `AgreementID` is set. When a recently-expired agreement is found (within 30 days), a warning log is emitted. Nil-safe: when no provider is wired, the engine proceeds with default operator cost. | SVC-06, STORY-071, `internal/operator/sor/engine.go` |
| Network Slice | Isolated virtual network within 5G | Dedicated resources for specific use case (e.g., IoT slice) |
| DNN | Data Network Name | 5G equivalent of APN |

## Argus Platform Terms

| Term | Definition | Context |
|------|-----------|---------|
| Tenant | An isolated enterprise customer account in Argus | Multi-tenant isolation unit |
| Operator Adapter | Pluggable module connecting Argus to a specific MNO | System-level, shared across tenants |
| Operator Grant | Permission for a tenant to use a specific operator adapter | Tenant ↔ Operator access control |
| Operator SoR Simulator | Passive HTTP server (`cmd/operator-sim`, container `argus-operator-sim`) that emulates the HTTP backends operators (Turkcell, Vodafone_TR, Turk_Telekom) would expose to an OSS integrator like Argus. Unlike `cmd/simulator` (which generates RADIUS/Diameter/SBA AAA traffic AS a SIM-fleet client), `cmd/operator-sim` serves AS the operator backend for argus's HTTP adapter to probe via the `http` sub-key in `adapter_config`. See STORY-089. | STORY-089, `internal/operatorsim/`, `cmd/operator-sim/` |
| Policy Version | Immutable snapshot of a policy rule set. States: `draft` (editable), `active` (in use, one per policy), `superseded` (replaced by newer active version), `rolling_out` (staged rollout, STORY-025), `rolled_back` (reverted, STORY-025), `archived` (manually deactivated). | SVC-05, STORY-023, TBL-14 |
| Policy Scope | Classification of policy applicability: `global` (tenant-wide), `operator` (per-operator), `apn` (per-APN), `sim` (per-SIM). Determines evaluation precedence per BR-4: SIM-specific > APN-level > operator-level > tenant default. | SVC-05, STORY-023, TBL-13 |
| Version State Machine (Policy) | Governs policy version lifecycle: draft → active (supersedes all previous active versions for same policy), active → superseded (when newer version activated), draft/active → archived (manual deactivation). Only draft versions can be edited or activated. Activation uses SELECT FOR UPDATE for race safety. FIX-231 adds DB-level enforcement: `policy_active_version` partial unique index on (policy_id) WHERE state='active' — at most one active version per policy at any time. `CompleteRollout` supersedes prior actives FIRST then activates target (prevents 23505 on the index). | SVC-05, STORY-023, FIX-231, TBL-14 |
| Policy Soft-Delete | Setting policy state to `archived` instead of physical row deletion. Blocked by `HasAssignedSIMs` EXISTS check if any SIMs reference the policy's versions. Archived policies are read-only but viewable for audit. | SVC-05, STORY-023 |
| Policy DSL | Domain-specific language for defining policy rules | e.g., `IF usage > 1GB THEN throttle_to(256kbps)` |
| Staged Rollout | Gradual policy deployment in configurable stages (default: 1% → 10% → 100%). Each stage selects SIMs randomly (`ORDER BY random() FOR UPDATE SKIP LOCKED`), updates `policy_assignments` (TBL-15), sends CoA for active sessions, publishes progress via NATS `policy.rollout_progress`. Stages >100K SIMs run as background jobs. Rollback reverts all SIMs to previous version with mass CoA. FIX-231 adds DB-level enforcement: `policy_active_rollout` partial unique index on `policy_rollouts(policy_id)` WHERE state IN ('pending','in_progress') — at most one active rollout per policy; concurrent StartRollout races produce 23505 which the service layer maps to `ErrRolloutInProgress` (422 `ROLLOUT_IN_PROGRESS`). | SVC-05, STORY-025, API-096..099, TBL-15/16, F-035, FIX-231 |
| Policy Rollout | Record in TBL-16 (`policy_rollouts`) tracking a staged policy deployment. Contains stage definitions (percentages), current stage index, state (`in_progress`/`completed`/`rolled_back`), migrated/total SIM counts, and error log. Created by `StartRollout()`, advanced by `AdvanceRollout()`. | SVC-05, STORY-025, TBL-16 |
| Stuck Rollout Reaper | Background cron job (`JobTypeStuckRolloutReaper`, schedule `*/5 * * * *`) added in FIX-231. Finds policy rollouts where `migrated_sims >= total_sims AND age > ARGUS_STUCK_ROLLOUT_GRACE_MINUTES` (default 10) and calls `CompleteRollout` for each, atomically transitioning the rollout to `completed` and activating the target policy version. Emits `policy.rollout_progress` event with state='completed' per reaped row. Prevents rollouts from remaining permanently `in_progress` due to race conditions or restarts. | SVC-09, FIX-231, `internal/job/stuck_rollout_reaper.go` |
| sims_policy_version_sync (Trigger) | PostgreSQL trigger function `sims_policy_version_sync()` added in FIX-231 (AC-2). Fires AFTER INSERT/UPDATE OF policy_version_id/DELETE on `policy_assignments` FOR EACH ROW. On INSERT/UPDATE: sets `sims.policy_version_id = NEW.policy_version_id` for the matching `sim_id`. On DELETE: sets `sims.policy_version_id = NULL`. The trigger is the **sole writer** to `sims.policy_version_id` — direct writes from application code are prohibited. Guarantees RADIUS hot-path consistency without requiring a JOIN on `policy_assignments`. | FIX-231, TBL-15, DEV-346 |
| Policy Assignment | Per-SIM policy version mapping in TBL-15 (`policy_assignments`). Tracks which policy version is assigned to each SIM during and after a rollout. Enables concurrent policy versions during staged rollout (some SIMs on old version, others on new). | SVC-05, STORY-025, TBL-15, BR-4 |
| CoA Dispatch (Rollout) | Process of sending Change of Authorization messages to active sessions during a rollout stage or rollback. Executed per-SIM in batches of 1000. CoA failures are logged but do not halt the rollout (fault-tolerant). | SVC-04/SVC-05, STORY-025, F-006 |
| Dry-Run | Policy simulation that evaluates a version against the SIM fleet without applying changes. Returns `total_affected_sims`, breakdowns by operator/APN/RAT, `behavioral_changes` (QoS/charging/access), and `sample_sims` with before/after. Sync (<100K SIMs) or async job (>100K). Cached 5min in Redis. | SVC-05, STORY-024, API-094, F-034 |
| Behavioral Change (Dry-Run) | Difference in policy evaluation outcome between current and candidate policy version for a SIM. Types: QoS upgrade/downgrade (bandwidth, priority), charging model change (prepaid/postpaid), access change (allow/deny). Detected by `DetectBehavioralChanges()`. | SVC-05, STORY-024 |
| Async Threshold | SIM count threshold (100K) determining whether a dry-run executes synchronously (HTTP 200) or asynchronously as a background job (HTTP 202 with job_id). Same pattern applies to staged rollout stage execution. | SVC-05/SVC-09, STORY-024, STORY-025 |
| SIM Fleet Filters | Dynamic query filter set derived from policy MATCH block conditions. Includes operator_id, APN names, RAT types, and optional segment_id. Used by dry-run and rollout to scope SIM queries. Built by `buildFiltersFromMatch()`. | SVC-05, STORY-024, store/sim.go |
| SIM Segment | Saved filter/group of SIMs based on criteria | Group-first UX navigation unit |
| Circuit Breaker | Pattern that disables a failing operator after N consecutive errors | Prevents cascade failures |
| WS Server | gorilla/websocket upgrade handler on `:8081/ws/v1/events`. JWT auth via query param (pre-upgrade) or first-message (5s timeout). Per-connection read/write pumps, ping/pong heartbeat (30s ping, 90s pong deadline configurable via `WS_PONG_TIMEOUT`), per-tenant max connections (default 100, configurable via `WS_MAX_CONNS_PER_TENANT`), per-user max connections (default 5, configurable via `WS_MAX_CONNS_PER_USER`), custom close codes (4001 unauthorized, 4002 max conns, 4003 auth timeout, 4004 internal error, 4029 per-user max). Graceful shutdown sends CloseGoingAway frames + `reconnect` control message. | SVC-02, STORY-040/060, `internal/ws/server.go` |
| WS Hub | Central WebSocket connection registry keyed by tenant_id. Methods: Register/Unregister, BroadcastAll/BroadcastToTenant, BroadcastReconnect, SubscribeToNATS (9 subjects via QueueSubscribe), ConnectionCount/TenantConnectionCount. EventEnvelope format: `{type, id, timestamp, data}`. Per-connection subscription filtering with wildcard support. Non-blocking send with 256-message buffer (drops oldest on overflow). | SVC-02, STORY-021/040/060, `internal/ws/hub.go` |
| WS Close Code | Application-defined WebSocket close codes: 4001 (Unauthorized -- JWT invalid/expired), 4002 (MaxConns -- tenant connection limit reached), 4003 (AuthTimeout -- no auth message within 5s), 4004 (InternalError), 4029 (MaxConnsPerUser -- per-user connection limit exceeded, oldest connection evicted). Standard codes: 1000 (NormalClosure), 1001 (GoingAway -- server shutdown). | SVC-02, STORY-040/060, DEV-133/DEV-152 |
| Metrics Collector | Central component (`internal/analytics/metrics/collector.go`) that records auth events in Redis and computes aggregated metrics (auth/s, error rate, latency percentiles). Uses Redis INCR counters (5s TTL) for rate tracking and ZSET (60s sliding window) for latency recording. Implements `MetricsRecorder` interface. | SVC-07, STORY-033 |
