# Bug Patterns & Prevention Rules

Runtime knowledge base of bugs that have occurred and rules to prevent them.
Read by: Planner (warnings), Gate/Scouts (compliance check), Developer (awareness).

## Patterns

- [2026-04-17] PAT-001 [STORY-084]: Double-writer on abort/session-end metric vectors ‚Äî Root Cause: Metric increment lives in both the low-level client (on error return) AND the high-level engine (on classified session outcome); a single logical abort double-counts when both layers fire. ‚Äî Prevention: For every session-level counter (e.g. `*SessionAbortedTotal`, `*SessionFailureTotal`), designate exactly one writer (typically the engine / outermost orchestrator); inner layers return wrapped sentinel errors and do NOT touch counters. Document the invariant in the package's `doc.go`. Tests must assert the TOTAL across reason buckets for one logical abort equals exactly 1. ‚Äî Affected: metrics layer (any protocol client + engine pair).
- [2026-04-17] PAT-002 [STORY-085]: Single-clock polling loop that misses a dynamically-shortened deadline ‚Äî Root Cause: An interim/polling loop driven by a fixed `ticker` (N-second interval) cannot honour a deadline shorter than N seconds away because the loop only wakes on the ticker. When an external event (e.g. CoA-Request) shortens the session deadline to a value < N, the session overshoots by up to N seconds. Discovered as STORY-085 Gate F-A6. ‚Äî Prevention: Use a composite `select` with both the periodic `ticker.C` and a `time.Timer.C` that is armed (or re-armed) to the effective deadline whenever that deadline is updated. The timer fires exactly when the deadline is reached; the ticker continues to drive interims. Stop and drain the timer in the session's `defer` block to prevent goroutine leaks. Covered by `TestSessionTimeout_SubIntervalDeadlineFires` (VAL-027). ‚Äî Affected: any engine or worker with an interim/polling loop whose deadline is externally mutable (CoA, policy update, admin override).
- [2026-04-18] PAT-003 [STORY-090 Gate F-A1]: Metric label set not expanded when write-fanout grows along a new dimension ‚Äî Root Cause: A gauge keyed on `{operator_id}` was written concurrently from N per-protocol goroutines; last writer wins and all-but-one protocol's state was silently discarded. Discovered when AC-10 required per-protocol gauge series. ‚Äî Prevention: Expand the metric label set BEFORE introducing the write-fanout, not after. Add a dimension label (e.g. `protocol`) the same commit that adds per-dimension goroutines/writers. The rename is a breaking change ‚Äî update Grafana dashboard expr + Prometheus alert rule + legend format atomically. ‚Äî Affected: any metric with fan-out writers that produces a `last-write-wins` semantic on a single label-set key. Covered by (VAL-028), `TestRegistry_SetOperatorHealth` + `TestHealthChecker_FansOutPerProtocol`.
- [2026-04-18] PAT-004 [STORY-090 Gate F-A5]: Goroutine cardinality N_parent √ó M_child per scheduler tick ‚Äî Root Cause: HealthChecker spawned one goroutine per (operator, protocol) per 30-second tick, producing up to 500 goroutines for 100 operators √ó 5 protocols. Discovered in Gate analysis (F-A5). ‚Äî Prevention: When a parent entity has M children and all M perform periodic work with the same interval, collapse to one ticker per parent that iterates children sequentially inside the goroutine. Only introduce per-child goroutines when child probes are independently timeable or when M √ó probe_duration ‚â• interval. Covered by (VAL-031).
- [2026-04-18] PAT-005 [STORY-090 Gate F-A2]: Masked-secret sentinel persisted via PATCH round-trip ‚Äî Root Cause: GET API emitted `"****"` sentinel for secret fields in `adapter_config`. A subsequent PATCH with the same payload (unchanged secrets) would silently persist `"****"` as the literal stored secret value, breaking all downstream auth. ‚Äî Prevention: Before encrypting and storing a PATCH body, splice the stored plaintext back into any field whose new value equals the sentinel. Additionally, if the stored envelope is undecryptable AND the body carries a sentinel, return 422 (`masked_sentinel_without_stored`) rather than silently persisting garbage. Apply the restore step atomically with validation/encrypt ‚Äî never mutate a secret to `"****"`. ‚Äî Affected: any API that emits a redaction sentinel on GET and accepts the same DTO shape on PATCH/PUT. - [2026-04-20] PAT-007 [FIX-202 Gate F-A1]: Mutex alone does not enforce happens-before ordering between goroutines for a shared map ‚Äî Root Cause: In `internal/api/dashboard/handler.go`, the `active_sessions` merge into the operator-health map was placed inside the operator-health goroutine body, protected by a mutex. However, the session-stats goroutine (which produces `sessionStatsByOp`) ran concurrently, so the merge could execute before `sessionStatsByOp` was populated ‚Äî the mutex prevented a data race on the map object itself but could not prevent the read-before-write logic error (intermittent nil map ‚Üí silently dropped `active_sessions`). ‚Äî Prevention: In a goroutine graph where goroutine A writes a map and goroutine B must read it, the read MUST happen after a synchronization point that guarantees A has completed (e.g. `wg.Wait()`, channel receive, or structured pipeline). Placing the read inside its own mutex-protected goroutine is insufficient when the writer runs concurrently and may not have completed. Covered by `go test -race ./internal/api/dashboard/...` (FIX-202 gate).

- [2026-04-20] PAT-008 [FIX-203]: Threshold-based delta triggers with first-value cold-start ‚Äî use a sentinel (0 or nil) to suppress the trigger on the first tick; only compare when both prev and curr are valid data. Prevents startup noise and divide-by-zero errors on transient timeouts. ‚Äî Root Cause: When a new `lastLatency map[healthKey]int` entry is seeded with 0 on operator registration, the first probe returns a real latency (e.g. 200ms). Without a cold-start guard, `|200-0|/max(0,1) = 200 = 20000%` would always exceed the 10% threshold and fire an event on every first tick ‚Äî flooding NATS on startup. The fix: gate the latency-delta branch with `prevLatency > 0 && currLatency > 0`. Cold start (`prevLatency == 0`) falls through without publishing; the second tick sees two real values and begins comparing. The status-flip path is unaffected (no cold-start issue there). ‚Äî Prevention: For any threshold-based delta trigger (`abs(curr - prev) / prev > threshold%`): (a) initialize the `prev` accumulator to a sentinel value (0 for int, nil for pointer) distinct from any real measurement; (b) guard the delta comparison with `prev > sentinel && curr > sentinel`; (c) add an explicit test case `TestCheckOperator_SuppressesOnColdStart` seeding `prev=0, curr=200` and asserting the event is NOT published. Apply the same pattern to any periodic comparator that uses the first sample as a baseline (rate calculators, SLA breach checks, anomaly detectors). ‚Äî Affected: `internal/operator/health.go` `HealthChecker.checkOperator` latency-delta gate at `:412-437`; any future per-entity delta comparator (auth_rate drift, session-count change alerting). Covered by `TestCheckOperator_SuppressesSubThresholdLatency` + `TestCheckOperator_StatusFlipStillPublishes` (FIX-203 gate, 3278 PASS).

- [2026-04-20] PAT-010 [FIX-205]: **Single-flight pattern for dependency-triggered refresh ‚Äî module-level mutex + failedQueue prevents N parallel refreshes from the same expiry trigger.** ‚Äî Root Cause: A page that fires N concurrent API calls on mount will receive N 401 responses if the token has expired. The equality check `if iface != nil` evaluates to TRUE because only BOTH halves being nil makes the interface comparable to the untyped `nil`.
