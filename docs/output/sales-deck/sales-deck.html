Temel Özellikler

  • CoA/DM: Gerçek zamanlı oturum değişikliği ve bağlantı kesme
  • Session Mgmt: SIM başına yapılandırılabilir eşanlı oturum limiti
  • Active-Active HA: Kesintisiz küme, tek hata noktası yok
  • Circuit Breaker: Operatör başına retry, dead letter queue
  • Network Slice: 5G SA dilim kimlik doğrulaması
  • Protocol Bridge: Diameter ↔ RADIUS köprüleme

Hot Path Performans (p99 < 50ms)

RADIUS Request → UDP listener (goroutine pool) → Parse packet ~0.1ms → Redis: cache lookup ~0.5ms → Policy evaluate ~0.1ms → IMSI route ~0.01ms → Operator forward ~5-30ms → Redis: session update ~0.5ms → NATS: accounting async → Response ~0.1ms

Staged Rollout (Canary Deployment)

1. Dry-Run → 2. Canary %1 → 3. %10 → 4. %100
  • Dry-run: "Bu kural 2.3M SIM'i etkiler" önizlemesi
  • Stage 1: 23K SIM'e yeni versiyon + CoA
  • Stage 2: Metrik inceleme sonrası 230K SIM
  • Stage 3: Tam yaygınlaştırma (2.3M)
  • Rollback: Her aşamada anında geri alınabilir + CoA

Policy Versioning

  • Her politika versiyonlanır (DRAFT → ACTIVE → ROLLING_OUT → ROLLED_BACK)
  • Concurrent versions: rollout sırasında birden fazla versiyon aktif
  • Operatör bazında özel politika yapılandırması
  • Değerlendirme sırası: SIM > APN > Operatör > Tenant default

Diğer platformlarda yok: Dry-run + staged rollout + rollback + DSL

Turkcell kesintisinde otomatik geçiş:

Turkcell RADIUS ✗ → Circuit breaker tetiklenir → Durum: DEGRADED → Alarm: email+Telegram → Per-SIM failover: "fallback-to-next" → Vodafone'a yönlendir → Turkcell toparlanır → Circuit breaker reset → SLA raporu üretilir
